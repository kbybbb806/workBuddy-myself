# 长期记忆

## 用户身份与偏好

### 职业与身份
- **职业**：摄影师
- **需求特点**：考试/出差 + 游玩结合的攻略
- **偏好**：文艺打卡点、小众景点、人少出片好的地方

### 旅行偏好
- 不喜欢人太多的热门景点
- 注重拍摄效果（光线、构图、氛围）
- 性价比优先
- 时间规划要合理（考试/工作优先）

### 技术偏好
- 攻略需要在手机上查看
- 导航要便捷（直接打开APP）
- 页面要美观、信息清晰

## 项目约定

### 旅行攻略规范
1. **页面结构**：
   - 响应式设计，手机优先
   - 时间线式行程展示
   - 每个地点包含导航按钮
   - 包含拍摄Tips和费用预估

2. **高德导航集成**：
   - 使用APP深层链接（amapuri:// / iosamap://）
   - 点击后3秒自动返回页面顶部
   - 显示"已打开高德地图"提示

3. **内容要素**：
   - 行程概览卡片
   - 详细时间线
   - 景点特色描述
   - 拍摄建议
   - 费用预估
   - 实用信息（地址、电话、营业时间）

### 文件命名规范
- 攻略页面：`{城市}游玩攻略.html`
- 酒店页面：`{城市}酒店推荐.html`

### 部署方式
- 使用GitHub Pages托管
- 仓库设为Public以启用Pages
- 推送后自动部署
